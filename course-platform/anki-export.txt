What is <b>UMPIRE Framework</b>?	A 6-step systematic problem-solving methodology used to break down coding interview questions before jumping into code: - **U**nderstand: Ask clarifying questions and define edge cases and sample inputs/outputs. - **M**atch: Identify pattern similarities (e.g., Two Pointers, Hash Map, Sliding Window). - **P**lan: Write out a step-by-step plain English algorithm or pseudocode. - **I**mplement: Translate the written plan into clean JavaScript code. - **R**eview: Line-by-line trace your solution with a test case to catch bugs. - **E**valuate: State time and space complexity using Big O notation.	glossary core-concepts dsa-javascript
What is <b>Big O Notation ($O$)</b>?	A metric measuring how the execution time or auxiliary memory of an algorithm scales as input size ($n$) grows toward infinity, ignoring hardware speeds and constants.<br><br><b>$O(1)$ Constant Time</b>: Execution time is flat and immediate regardless of input size (e.g., array index lookup `arr[0]`, `.push()`, Map/Set lookups).<br><b>$O(n)$ Linear Time</b>: Execution time grows proportionally with input size (e.g., single `for...of` loop, `.map()`, `.filter()`, `.indexOf()`).<br><b>$O(n^2)$ Quadratic Time</b>: Execution time grows proportional to the square of input size (e.g., nested loops checking every pair in an array).	glossary core-concepts dsa-javascript
What does <b>$O(1)$ Constant Time</b> mean?	Execution time is flat and immediate regardless of input size (e.g., array index lookup `arr[0]`, `.push()`, Map/Set lookups).	glossary core-concepts dsa-javascript
What does <b>$O(n)$ Linear Time</b> mean?	Execution time grows proportionally with input size (e.g., single `for...of` loop, `.map()`, `.filter()`, `.indexOf()`).	glossary core-concepts dsa-javascript
What does <b>$O(n^2)$ Quadratic Time</b> mean?	Execution time grows proportional to the square of input size (e.g., nested loops checking every pair in an array).	glossary core-concepts dsa-javascript
What is <b>Space Complexity</b>?	The auxiliary RAM/memory allocated by an algorithm during execution ($O(1)$ in-place modification vs $O(n)$ creating new arrays/objects).	glossary core-concepts dsa-javascript

What is <b>Cache Key</b>?	A string that uniquely identifies a cached directory. Typically built from the runner OS and a hash of a lockfile (e.g., `package-lock.json`).
