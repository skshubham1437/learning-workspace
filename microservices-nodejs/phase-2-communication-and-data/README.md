# Phase 2: Communication & Data (~2.5 weeks)

## Modules

### Module 03 — Communication Patterns
Synchronous: REST vs GraphQL vs gRPC. Asynchronous: Event-driven architecture, message queues (RabbitMQ, Kafka, Redis Streams, NATS).

### Module 04 — Data Management
Database-per-service. Shared database anti-pattern. CQRS. Event Sourcing. Materialized Views. Data replication. Eventual consistency.

### Module 05 — Distributed Transactions
Why ACID breaks. Saga Pattern (Choreography vs Orchestration). Transactional Outbox. Inbox Pattern. Idempotency keys. Failure recovery.

## Goal
By the end of this phase, you should be able to design how services talk to each other and manage their data independently — the two hardest problems in distributed systems.
