# Phase 5: Deployment & Infrastructure (~3 weeks)

## Modules

### Module 10 — Deployment
Docker in production (not just local). Kubernetes fundamentals through the lens of deploying services. Helm charts. Rolling deployments. Health checks and readiness probes.

### Module 11 — Service Mesh
What a service mesh actually does. Istio vs Linkerd. When it's overkill. Traffic management, observability, and security at the mesh layer.

### Module 16 — AWS Cloud Patterns
ECS vs EKS vs Lambda: when to use each. SQS/SNS for async communication. EventBridge for event routing. API Gateway (AWS-managed). CloudWatch for logs and metrics. X-Ray for distributed tracing. DynamoDB streams. IAM best practices for microservices. Deploying the capstone services to ECS with SQS.

## Goal
By the end of this phase, you should be able to take a set of Node.js microservices from "works on my machine" to "running in production on Kubernetes with proper rollout strategies" — and understand how the same architecture maps onto AWS managed services.
