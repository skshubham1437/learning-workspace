# Mission: React From Basics To Production Confidence

## Why
You want to move beyond React basics and become comfortable reading, changing, and eventually designing real React application code, especially the `xley-frontend` project. The goal is to learn the concepts along the way, then use those concepts to confidently trace features, understand state and data flow, and choose packages like TanStack Query or Redux Toolkit when the codebase actually needs them.

## Success looks like
- Read a React feature and explain its component tree, data flow, local state, server state, and side effects.
- Make small changes in a production React codebase without guessing where the logic lives.
- Decide whether a problem belongs in React state, URL state, server cache, context, Redux, or a custom hook.
- Understand common advanced patterns: composition, reducers, context, effects, custom hooks, routing, data fetching, performance, testing, and forms.
- Explain each important React concept in plain language before applying it in code.
- Use package documentation directly and map it back to existing project code.

## Constraints
- Starting point: you know React at a high level, including basics like what React is and the virtual DOM.
- Lessons must not assume concept knowledge; each new concept should be introduced, named, practiced, and then connected to real code.
- Lessons should use real code-reading practice whenever the `xley-frontend` path is available.

## Out of scope
- React Native.
- Deep React internals or compiler implementation unless they become necessary for debugging.
- Next.js, Remix, or other React frameworks unless `xley-frontend` uses them.
