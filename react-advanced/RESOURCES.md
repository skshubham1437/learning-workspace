# React Advanced Resources

## Knowledge

- [React Learn: Quick Start](https://react.dev/learn)
  Official React docs for the daily core: components, JSX, props, state, events, lists, and hooks. Use for: rebuilding fundamentals without stale tutorial habits.
- [React Learn: Thinking in React](https://react.dev/learn/thinking-in-react)
  Official method for breaking UI into components, finding minimal state, and placing ownership. Use for: reading and designing features.
- [React Learn: State as a Snapshot](https://react.dev/learn/state-as-a-snapshot)
  Official explanation of why state reads behave like snapshots during a render. Use for: debugging stale values and event handlers.
- [React Learn: Preserving and Resetting State](https://react.dev/learn/preserving-and-resetting-state)
  Official guide to how component position and keys affect state lifetime. Use for: forms, tabs, lists, and surprising resets.
- [React Learn: You Might Not Need an Effect](https://react.dev/learn/you-might-not-need-an-effect)
  Official guidance on avoiding unnecessary effects. Use for: derived data, event logic, and reducing accidental complexity.
- [React Reference](https://react.dev/reference/react)
  Official API reference for hooks and React APIs. Use for: precise behavior after the idea is understood.
- [TanStack Query React Docs](https://tanstack.com/query/latest/docs/framework/react/overview)
  Official docs for server-state caching and synchronization. Use for: API data, loading/error states, invalidation, and background refetching.
- [Redux Toolkit Quick Start](https://redux-toolkit.js.org/tutorials/quick-start)
  Official Redux Toolkit entry point. Use for: app-wide client state with structured updates, especially when local state and context become awkward.
- [React Router Routing Docs](https://reactrouter.com/start/data/routing)
  Official React Router docs. Use for: route trees, nested layouts, URL params, loaders, and navigation.
- [Vite Guide](https://vite.dev/guide/)
  Official Vite docs. Use for: understanding modern React dev servers, build entry points, environment variables, and project structure.

## Wisdom (Communities)

- [React Community page](https://react.dev/community)
  Official index of React communities and support channels. Use for: choosing where to ask code-level questions or discuss architecture.
- [Reactiflux](https://www.reactiflux.com/)
  Large React-focused Discord community. Use for: testing architectural judgment and seeing how experienced React developers discuss tradeoffs.
