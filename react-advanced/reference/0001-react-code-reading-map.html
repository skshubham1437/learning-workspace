Reference 01

React Code Reading Map

A compact checklist for reading unfamiliar React features without getting lost.

The three questions

What renders?

Find the route, page, component tree, and repeated list boundaries.

What changes?

Separate props, local state, derived values, URL values, and server data.

Who owns it?

Find the component, hook, cache, store, or router responsible for updates.

The reading order

1Start at the user-visible screen or route.
2Identify the top-level page component and its immediate children.
3Mark every prop flowing into the feature.
4Mark every `useState`, reducer, context, query, router hook, and custom hook.
5For every displayed value, decide whether it is original state or derived data.
6For every event handler, trace which owner it updates.
7Only after the above, inspect styling, performance, and package details.
Rule of thumb: if a value can be calculated from props or existing state during render, treat it as derived data first. Reach for state only when the app must remember a changing value.

Primary source: React docs, Thinking in React. Related lesson: Read a React Feature From the Outside In.
