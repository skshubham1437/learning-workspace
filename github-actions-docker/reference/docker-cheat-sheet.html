Docker Cheat Sheet

Core CLI commands, Dockerfile instructions, and mental models in one place.

Containers

CommandWhat it does
docker run -d -p 8080:80 --name web nginxPull + create + start a container. -d = detached, -p = publish port host:container
docker psList running containers
docker ps -aList all containers including stopped ones
docker stop [name]Gracefully stop a container (SIGTERM → SIGKILL after 10s)
docker rm [name]Delete a stopped container
docker rm -f [name]Force-stop and delete a running container
docker logs -f [name]Stream live stdout/stderr from a container
docker exec -it [name] sh"SSH" into a running container. Use bash if sh isn't available.
docker inspect [name]Full JSON metadata: IP, mounts, env vars, network config

Images

CommandWhat it does
docker build -t my_app:v1 .Build image from Dockerfile in current dir. -t = tag (name:version)
docker imagesList all local images with size and tag
docker pull postgres:15Download an image without running it
docker rmi [name]Delete a local image
docker image pruneDelete all dangling (untagged) images
docker history [name]Show layer-by-layer breakdown of an image and its sizes

Volumes

CommandWhat it does
docker volume create my_dataCreate a named volume (Docker manages its location)
docker volume lsList all volumes
docker volume rm my_dataDelete a volume (only works if no container is using it)
docker volume pruneDelete all volumes not attached to any container. Irreversible.
-v [source]:[destination]General syntax: Maps host volume or path (left) to container path (right)
-v my_data:/var/lib/postgresql/dataMount named volume my_data to container path /var/lib/postgresql/data
-v $(pwd):/appBind mount current host directory into container /app (for dev hot-reload)

Networks

CommandWhat it does
docker network create my_netCreate a user-defined bridge network
docker network lsList all networks
docker network inspect my_netSee which containers are connected and their internal IPs
--network my_netConnect a container to a network at run time
docker network connect my_net [container]Connect an already-running container to a network

Dockerfile Instructions

InstructionWhen it runsWhat it does
FROM node:20-alpineBuildBase image — the starting layer. Always first.
WORKDIR /appBuildSets the working directory for all subsequent instructions
COPY package.json .BuildCopies files from host into the image layer
RUN npm installBuildExecutes a command and saves the result as a new layer
ENV NODE_ENV=productionBuild + RuntimeSets an environment variable available at build and run time
EXPOSE 3000Documentation onlyDocuments which port the app listens on. Does not publish it.
CMD ["npm", "start"]RuntimeDefault command when container starts. Can be overridden at runtime.
ENTRYPOINT ["node"]RuntimeFixed command. CMD becomes its arguments. Harder to override.

Mental Models

ConceptAnalogy
ImageA class definition (blueprint). Immutable. Can be instantiated many times.
ContainerA running instance of a class. Ephemeral. Disposable.
Named VolumeAn external hard drive Docker manages for you. Outlives the container.
Bind MountA shared folder between your host and the container.
NetworkA private LAN where containers can find each other by name (DNS).
EXPOSEA sticky note on a box saying "port 80 is inside here". Doesn't open anything.
-p 8080:80The actual delivery — punches a hole from your host 8080 to the box's 80.
