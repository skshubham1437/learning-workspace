# Mission: CI/CD with GitHub Actions & Docker

## Why

To become a well-rounded developer who can confidently ship code — not just write it. Understanding CI/CD and containerisation means you can set up automated testing, build reproducible environments, and deploy any project (React frontend, Node/Express API, Go backend) without depending on someone else to "do the DevOps."

## Success looks like

- Can write a GitHub Actions workflow from scratch that lints, tests, and builds a project on every push
- Can write a multi-stage Dockerfile for each of the three stacks (React, Node/Express, Go)
- Can wire Docker builds into a GitHub Actions pipeline that pushes images to a container registry
- Understands the *why* behind each piece — not just copy-pasting YAML

## Constraints

- Learning for conceptual depth, not shipping to production right now
- Has seen Docker/Actions mentioned but never set one up hands-on
- Prefers building real (small) example apps to anchor the learning

## Out of scope

- Kubernetes orchestration (for now)
- Cloud-specific deployment targets (AWS ECS, GCP Cloud Run, etc.)
- GitLab CI, Jenkins, or other CI systems
- Advanced security hardening (OIDC, supply-chain signing)
