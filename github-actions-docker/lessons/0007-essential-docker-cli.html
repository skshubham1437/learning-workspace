Lesson 07

Essential Docker CLI

Mastering the terminal commands you need to control containers every single day.

📖 Primary source: Docker CLI Reference

The Docker Run Command

The most important command in Docker is docker run. It downloads an image (if you don't have it locally), creates a container, and starts it.

# The basic anatomy
docker run [OPTIONS] IMAGE_NAME [COMMAND]

Crucial Flags

  • -d (Detached): Runs the container in the background, freeing up your terminal.
  • -p 8080:80 (Port mapping): Forwards traffic from your host's port 8080 to the container's port 80.
  • --name my_web: Gives your container a readable name instead of a random string like angry_einstein.
  • -e MYSQL_ROOT_PASSWORD=secret (Environment variable): Injects environment variables directly into the container.
  • -it (Interactive TTY): Usually paired with a shell command (like sh or bash) to give you an interactive terminal inside the container.

Managing Containers

Once a container is running (or stopped), you need to manage it.

# 1. List RUNNING containers
docker ps

# 2. List ALL containers (including stopped ones)
docker ps -a

# 3. Stop a running container gracefully
docker stop [CONTAINER_NAME_OR_ID]

# 4. Remove a stopped container (frees up disk space)
docker rm [CONTAINER_NAME_OR_ID]

# 5. Remove a RUNNING container by force (stop + rm)
docker rm -f [CONTAINER_NAME_OR_ID]

Debugging Containers

When things go wrong (and they will), these are your two best friends:

# View the standard output (console logs) of a container
docker logs my_web

# Stream the logs live (like tail -f)
docker logs -f my_web

# Execute a command INSIDE an already running container
# (This is how you "SSH" into a running container)
docker exec -it my_web sh

Your Turn

Let's spin up a completely isolated Nginx web server on your machine in 10 seconds.

  1. Run this command: docker run -d -p 8080:80 --name my_first_server nginx:alpine
  2. Open your browser and go to http://localhost:8080. You should see "Welcome to nginx!".
  3. Check that it's running: docker ps
  4. Check its logs: docker logs my_first_server
  5. "SSH" into it: docker exec -it my_first_server sh
  6. While inside, look around (ls /usr/share/nginx/html), then type exit.
  7. Stop and delete the server: docker rm -f my_first_server
