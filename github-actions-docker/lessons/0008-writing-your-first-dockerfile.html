Lesson 08

Writing your first Dockerfile

Turning your application into an immutable container image.

📖 Primary source: Dockerfile Reference

What is a Dockerfile?

If a container is a running environment, an image is the blueprint for that environment. A Dockerfile is just a text file containing the instructions to build that blueprint.

Images are built in layers. Every command in a Dockerfile creates a new layer on top of the previous one. Docker aggressively caches these layers to make builds faster.

The 4 Essential Instructions

You can build 90% of the world's applications using just these four Dockerfile instructions:

1. FROM

Every Dockerfile MUST start with a FROM instruction. It defines your base image (the starting point).

FROM node:20-alpine

2. COPY

Copies files from your host machine (your Mac) into the image.

# Format: COPY [host-path] [container-path]
COPY package.json /app/
COPY . /app/

3. RUN

Executes a command DURING THE BUILD PROCESS. This is typically used to install dependencies or compile code. The result is saved into the image.

RUN npm install

4. CMD

Defines the default command to execute WHEN THE CONTAINER STARTS. There can only be one CMD instruction in a Dockerfile.

CMD ["npm", "start"]

Putting It Together

Here is what a complete, basic Dockerfile for a Node.js app looks like:

# 1. Start with a lightweight Node.js environment
FROM node:20-alpine

# 2. Set the working directory inside the container
WORKDIR /app

# 3. Copy package.json first (for caching optimization)
COPY package.json .

# 4. Install dependencies (runs during 'docker build')
RUN npm install

# 5. Copy the rest of the source code
COPY . .

# 6. Expose the port the app runs on (mostly for documentation)
EXPOSE 3000

# 7. Start the app (runs during 'docker run')
CMD ["npm", "start"]

Building the Image

Once you write a Dockerfile, you build it using the docker build command.

# -t tags the image with a name (my_node_app)
# The . at the end tells Docker to look for the Dockerfile in the current directory
docker build -t my_node_app .

Your Turn

Let's build a tiny Python web server from scratch!

  1. On your computer, create a new folder: mkdir docker-test && cd docker-test
  2. Create a file named index.html with the text <h1>Hello from my custom image!</h1>.
  3. Create a file named Dockerfile (no extension).
  4. Inside the Dockerfile, write these three lines: 
    FROM python:3.9-alpine
WORKDIR /app
COPY index.html .
CMD ["python", "-m", "http.server", "8000"]
  5. Build your image: docker build -t my_python_server .
  6. Run your brand new custom image: docker run -d -p 8080:8000 my_python_server
  7. Visit http://localhost:8080 in your browser!
