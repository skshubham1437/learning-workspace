# Learner has seen CI/CD and Docker mentioned but never configured either

The user reports exposure at the "I've seen/read about them" level — meaning they can likely recognise terms like "workflow," "container," and "image" but have not written a Dockerfile or a `.yml` workflow file themselves. Future sessions should start from foundational concepts (what a pipeline *is*, what a container *is*) before jumping into syntax.

## Implications
- Lesson 1 should be a conceptual map — the "big picture" — before touching any YAML or Dockerfile.
- Avoid assuming familiarity with YAML syntax, Docker CLI commands, or GitHub's runner infrastructure.
- Lean heavily on analogies and visual models early on.
