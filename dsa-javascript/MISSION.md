# Mission: DSA and Problem Solving in JavaScript

## Why
To build strong problem-solving skills and master Data Structures and Algorithms (DSA) using JavaScript. The goal is to feel confident tackling LeetCode questions, performing well in technical interviews, and writing more efficient, optimized code in day-to-day work.

## Success looks like
- Can systematically break down a problem using a framework (like UMPIRE) instead of staring at a blank screen or jumping straight into coding.
- Can recognize common patterns (Two Pointers, Sliding Window, Fast & Slow pointers) rather than trying to memorize specific solutions.
- Can explain the time and space complexity (Big O) of written solutions.
- Can comfortably solve medium-level LeetCode problems in JavaScript within 30-45 minutes.

## Constraints
- The user feels "not good at problem solving skills" currently, meaning we must focus heavily on the *process* of solving a problem before throwing complex algorithms at them.
- Time must be spent on the "understanding" and "planning" phases of problem solving, not just syntax.
- Everything will be taught in modern JavaScript (ES6+).

## Out of scope
- Hard-level competitive programming tricks that rarely appear in standard interviews.
- Obscure data structures (e.g., Fibonacci heaps) that are not practical for general software engineering interviews.
- Memorizing hundreds of random LeetCode questions.
