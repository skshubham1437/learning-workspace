# DSA and Problem Solving Resources

## Knowledge

- [Educative: Grokking the Coding Interview Patterns](https://www.educative.io/courses/grokking-the-coding-interview)
  The gold standard for pattern-based learning. Use for: Understanding the ~20 core patterns (Sliding Window, Two Pointers, etc.) instead of memorizing individual problems.
  
- [NeetCode (Roadmap & Video Explanations)](https://neetcode.io/)
  An incredibly structured roadmap of 150 core LeetCode problems organized by pattern. Use for: High-quality video explanations and a structured list of practice problems.

- [The UMPIRE Interview Strategy](https://guides.codepath.org/compsci/UMPIRE-Interview-Strategy)
  CodePath's guide to the UMPIRE method (Understand, Match, Plan, Implement, Review, Evaluate). Use for: A strict, step-by-step framework on how to approach ANY problem when you don't know the answer immediately.

- [MDN Web Docs: JavaScript Reference](https://developer.mozilla.org/en-US/docs/Web/JavaScript)
  The canonical reference for JS. Use for: Looking up exact syntax for array methods (`.map`, `.reduce`, `.slice`, `.splice`) and Map/Set objects.

## Wisdom (Communities)

- [r/leetcode](https://www.reddit.com/r/leetcode/)
  Active community for interview prep. Use for: Understanding current interview trends, company-specific experiences, and moral support.

## Gaps

- Most resources throw algorithms at you without teaching *how to think*. We will lean heavily on the UMPIRE method early on to bridge this gap.
